• Explicación de la herencia aplicada

La clase Admin actúa como hija de la clase Usuario utilizando "class Admin extends Usuario", de esta manera
Admin hereda métodos y propiedades de Usuario por medio de los métodos getters y setters

• Diferencias entre Usuario y Admin

Usuario: 
 
 *Es la clase base
 
 *Tiene las propiedades de nombre y correo
 
 *Tiene métodos getters y setters

Admin:

 *Es una clase hija
 
 *Implementa el método getRol()
 
 *Accede a las propiedades por medio de get y set
